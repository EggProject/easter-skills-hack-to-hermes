# 08 — Migration Note Format (3-file split)

## Goal

Emit a deterministic, exhaustive, AI-readable migration note at the worktree root. The note is split into THREE files because the audiences are different:

1. **`MIGRATION.md`** — top-level index. Human- and AI-readable entry point. Links to the other two.
2. **`MIGRATION.hermes-patch.md`** — covers Script #1's patches (cap-raise + 7 Task E sites). Audience: operators running Script #1.
3. **`MIGRATION.skill-port.md`** — covers the migrated skill's per-binding replacements (T3 inventory). Audience: operators reviewing the skill body / eval pipeline.

All three are source-controlled at the worktree root. `--emit-migration-note` (on Script #1) regenerates #1 and #2. The skill-port note (#3) is generated by the migrated skill's own installer (or Script #1 with `--emit-skill-migration-note`, per Q5).

Script #3 (reporter) is STDOUT + `--json PATH` only — it does NOT write to the worktree. Script #3 has no `--emit-migration-note` flag.

```markdown
- `MIGRATION.hermes-patch.md` — Script #1 patches (cap raise + 7 Task E sites).
- `MIGRATION.skill-port.md` — migrated skill bindings (T3 inventory).

## How to apply

1. Run Script #1 against your user-owned Hermes checkout:
   `uv run hermes-skill-creator-patch --apply --task-e-redirect --target <hermes-checkout>`
2. Install the plugin and migrated skill:
   `uv run python -m hermes_skill_creator_plugin.install`
3. Run Script #2 to audit / flip all profiles:
   `uv run hermes-skill-creator-profiles --apply`
4. Run Script #3 to print a report of enabled skills (optional):
   `uv run hermes-skill-creator-report --profile default`
5. Start a fresh Hermes session. The new skill appears in `skills_list()`.
```

## Determinism

- `MIGRATION.hermes-patch.md` and `MIGRATION.md` are byte-identical across runs given the same `--target` (same `git_head`) and the same `--task-e-redirect` / `--no-schema-redirect` flags.
- `MIGRATION.skill-port.md` is byte-identical across runs given the same source tree and Hermes target.
- The "Generated at" timestamp uses a frozen value when the env var `HERMES_SKILL_CREATOR_FROZEN_TIME` is set; in CI it is always set; in production it defaults to the actual time.
- All table rows are sorted by site_id (lexicographic) for the patch file.
- All JSON keys in sidecar reports are sorted.
- No trailing whitespace; final newline; LF line endings.

## Exhaustiveness

- The T3 inventory row count in `MIGRATION.skill-port.md` MUST equal 18 (the number of rows in `07-skill-creator-migration.md`'s T3 table). Enforced by `test_migration_note_exhaustive_skill_port`.
- The patch-site row count in `MIGRATION.hermes-patch.md` MUST equal 1 (cap) + (7 or 6, depending on `--no-schema-redirect`) for Task E. Enforced by `test_migration_note_patch_rows_match_flags`.
